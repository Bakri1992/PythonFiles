Offensichtlich ist die Liste bei der linearen Suche schon ab 3 Buchstaben deutlich 
langsamer als das Set.

Warum? =>
Der Hauptvorteil der Verwendung eines Sets im Vergleich zu einer Liste besteht darin,
dass es eine hoch optimierte Methode zum Überprüfen enthält, ob ein bestimmtes Element
im Set enthalten ist. Dies basiert auf einer Datenstruktur, die als Hashtabelle bekannt
ist. Da Sets ungeordnet sind, können wir nicht auf Elemente mithilfe von Indizes 
zugreifen, wie wir es bei Listen tun.


Warum sind die Ergebnisse beim Set so komisch? Evtl. nochmal laufen lassen ?
